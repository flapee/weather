# Weather

## Summary

The MVP is a single-day weather chart with one shared 24-hour x-axis and one normalized y-space. It overlays weather metrics in one chart area instead of stacking separate bands. Raw values remain visible in the Trends panel, current scrubber readout, and event list because the vertical position is comparative only.

## Data

- Uses Open-Meteo hourly forecast data plus Open-Meteo air-quality data, with bundled sample data as a fallback.
- Defaults to Prague coordinates and keeps latitude/longitude editing in the settings panel.
- Fetches 30 past days plus today and 3 future forecast days, then normalizes each metric against the returned hourly range. Live data refreshes every 60 minutes while the page is visible, and stale data refreshes when the tab becomes visible, the window regains focus, or the device comes back online. Successful refreshes preserve the current day/week view when that range is still available and show the local update time in the data status.
- Air-quality values come from `air-quality-api.open-meteo.com/v1/air-quality` with `pm2_5`, `dust`, `alder_pollen`, `birch_pollen`, `grass_pollen`, `mugwort_pollen`, and `olive_pollen`. Pollen is CAMS Europe data and can be unavailable outside its supported region or season.
- After live data loads, the app fetches same-calendar-date air-quality history for the previous five years using `start_date` and `end_date`, then compares daily PM2.5 average, dust average, and peak pollen against the selected year.
- Displays the selected Open-Meteo day plus the next midnight point so the 24:00 endpoint is available. The default selected day is today, and the compact header shows the location with previous/next arrows flanking the day label; offsets beyond yesterday/tomorrow use the weekday. Clicking Today enters a 7-day chart mode covering today plus the next six days only when each day in that 7-day window has real weather coverage.

## Axes

- Time axis: daily mode uses `00`, `06`, `12`, `18`, `24`; 7-day mode uses weekday labels and day-boundary grid lines, except Sunday is replaced with `Today`, `Yesterday`, `Tmrw`, or an ordinal date.
- Left and right side labels are omitted so the plot can use the full chart width.
- The normalized y-space is not a unit axis; it is only a shared visual comparison space.

## Visual Encoding

- Conditions: weather pictograms above the chart distinguish sun and moon using hourly `is_day`; moon icons use the current lunar phase. All glyphs that use the cloud shape draw it reduced by 30% with a consistent vertical offset, and rain/snow/storm details are drawn inside that same scaled cloud group so the full glyph stays coherent. The repeated pictogram row uses two-hour windows at all layout sizes in daily mode and includes the final 23:00 hour. Sunrise and sunset use the actual weather-condition pictogram at the exact hourly x-position, shifted slightly closer to the chart, and suppress nearby regular condition icons when spacing would collide.
- Temperature: actual temperature is a continuous, smoothed, 30% opacity value-colored line; feels-like temperature is the stronger value-colored line. Only the feels-like daily high/low have subtle dots. Daily and 7-day modes use the same loaded-range normalization so the same temperature lands at the same relative height; in 7-day mode, temperature lines are 50% thinner and feels-like opacity is 50%.
- Rain probability: nonzero values are drawn as an inverted dotted blue line whose opacity follows the probability value, then receives a 0.6 chart opacity multiplier. When Rain is focused, the probability-level opacity multiplier is disabled but the 0.6 chart multiplier remains.
- Rain amount: nonzero daily values are inverted blue bars hanging down from the top of the plot at 40% opacity, centered in the reported hour interval. In 7-day mode, rain amount is summed into quarter condition-glyph windows and uses the same rain scale as daily mode, giving twelve 2-hour columns per day.
- Wind/gusts: smoothed fill between wind speed and gust speed, with no boundary lines. Wind and gust share one range, gust is clamped to at least wind speed for plotting, and a minimum pixel thickness preserves a visible area when gust data is missing or equal to wind. The fill is rendered as a smooth alpha field: previous vertical transparency is retained, with a center-weighted multiplier of `1.5` at the middle between wind/gust and `0.3` at either edge. Optional green-only contour shading is enabled in the raster renderer, modulating the wind color toward darker teal bands without introducing black shadows.
- Humidity: inverted smoothed fill only, using a green-to-blue gradient and no top edge line. Fill opacity is multiplied by humidity level across the whole area: below 75% trends to `0.8`, below 50% to `0.4`, below 25% to `0.2`, and 100% reaches `1.0`.
- Visibility: default trial mode uses a generated raster alpha mask from the visibility curve, cutting only into the lower humidity/wind fills. The center cuts at roughly 80% intensity and fades smoothly outward without an edge accent. Switching `visibilityMode` to `overlay` restores a single blurred visibility area.
- UV: the plot does not draw a UV curve. The axis shows the upward index-3 crossing as `>`, the peak UV value, and the downward index-3 crossing as `<`. Crossing markers sit on the bottom axis so the axis line bisects the symbol. If the two index-3 crossings are less than 2 hours apart, only the peak value is shown on the axis.
- Air quality: PM2.5, dust, and pollen are drawn as normalized lines inside only the bottom quarter of the plot. Zero-value air-quality points are not drawn. Nonzero line segments use a value-based opacity multiplier, matching the rain-probability treatment. When an air-quality metric is focused, the value-based opacity multiplier is disabled for that metric so the nonzero line remains clearly visible. PM2.5 is dotted and 10% transparent, dust is dashed, and pollen series use lighter dashed lines. Raw values are kept in the Trends panel and scrubber readout. Previous-year comparison lines are faint and follow the active air-quality metric, defaulting to grass pollen.
- Current-day history: on the current day only, past data is faded as a mask over all plotted data series. The opacity is full at the current time, ramps smoothly down over the previous 60 minutes, and stays at 30% opacity for older history.

## Interaction

- One scrubber controls all metrics. It is hidden by default and appears only while dragging the chart. Its vertical line is a subtle contour-like shadow, and scrubber dots use the matching chart color and opacity while omitting zero-valued rain-probability and air-quality points that are not drawn.
- Dragging or stepping the selected hour updates the scrubber and, while scrubbing, the Trends tile values. On touch screens, scrubbing initiates only after a long press inside the chart; until then, horizontal swipes can change days and vertical gestures can scroll. Once touch scrubbing is active, native text selection and page scrolling stay locked until the scrub disengages.
- Header date controls and page-wide left/right swipes move the selected day backward or forward while preserving the selected hour. In 7-day mode, the same sideways controls slide the 7-day window rather than leaving week mode, but cannot move into a 7-day range with missing weather days. The Today control opens the 7-day mode from Today; a horizontal pinch on a single-day chart opens a 7-day window starting on that viewed day, and a horizontal pinch-out / zoom-in gesture on the 7-day chart returns to Today. When the viewport rotates into landscape, the page scrolls to the chart while ensuring the bottom time labels remain inside the visible viewport.
- In 7-day mode, UV axis marks and Notable events are hidden. Trends remain visible, expand to the full details width, and their resting values summarize the full 7-day chart range; while scrubbing, they show the scrubbed hour values. The bottom day axis is anchored to the active 7-day window start; only Sunday is replaced with `Today`, `Yesterday`, `Tmrw`, or an ordinal date such as `13th`, while other labels remain weekdays. Weather condition glyphs use three 8-hour slots per day in a compact top band. A compact daily summary sits between the glyphs and plot edge, showing actual temperature low/high using the temperature color scale and bottom-aligning the text near the plot edge. The bottom label band keeps only a small safety margin below the time labels.
- Scrubber marks are plain dots on the active metric positions.
- The compact header shows current conditions with selected-day rain, severe weather, or AQ context inline to the right of the condition text. Thunderstorm risk is shown as ⛈️, downpour risk as 🌧️ with the peak rain amount, and probability-only rain with 0 mm is not promoted. PM2.5 appears only at 10 µg/m³ or higher, dust appears at 10 µg/m³ or higher, and strongest pollen appears at 15 𖡼/m³ or higher.
- The Trends panel lists only metrics that have visible data for the selected day. Metrics whose selected-day chart would contain only zero values are hidden, and UV remains drawn only as axis markers/events rather than as a trend tile.
- Trends merges the former metric controls and AQ history into three-column tappable tiles on narrow views. In both daily and 7-day modes, the grid may add columns when a three-column tile would become wider than a 3:1 width/height ratio. In landscape, Trends move to a narrow right-side column beside the chart with one characteristic per row; the history/range row is hidden, icons use the metric color, and the value side gets 55% of the tile with values aligned to the right edge. The Trends heading is hidden at narrow width. The top line uses condition-style icons by default; the focused tile swaps the icon for the full metric label. It shows the daily summary by default, switching to the scrubbed hour value while dragging the chart; values are bold, and units use the muted AQ-history unit style. The bottom line keeps the compact AQ-history range strip grammar at the same width as the top row, with min/max labels below the mini chart and the `<`/`>` trend marker centered on the mini chart: current daily value against recent daily values and range. Rain history uses daily rain amount, not rain probability, and visibility uses daily high visibility.
- The Trends panel supports hover/click highlighting.
- Clicking an already-selected trend tile clears selection. The just-cleared tile suppresses immediate hover refocus until the pointer leaves, so render-affecting focus multipliers return to their default state.
- The notable-events panel lists UV events, rain start, severe weather, peak downpour, peak gust, lowest visibility, peak dust, and peak pollen.

## Current Implementation Notes

- The prototype is a standalone `index.html` file.
- It uses inline SVG for all chart drawing.
- Smoothed curves use cubic segments between hourly points.
- If Open-Meteo fails or times out, the chart reverts to sample data.
